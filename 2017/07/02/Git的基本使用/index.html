提交本地代码

提交本地代码需要用到add 和 commit命令即可，add用于将想要提交的代码添加进来，commit则是真正地区执行提交操作。

  1. 添加 app 目录 或 单个文件
  1. 添加所有的文件
  1. 提交

在commit命令的后面，一个要通过 -m参数来加上提交的描述信息，不然以后连自己都不知道每次提交的意义。

如何忽略不想提交的文件

Git会检查代码仓库的目录下是否存在一个名为.gitignore的文件，如果存在的话，就会一行行读取这个文件的内容，并把每一行指定的文件或目录排除在版本控制之外，.gitignore中指定的文件或目录可以使用 “*” 通配符。

Git中分支的用法

分支
分支是版本控制中比较高级且比较重要的一个概念，它主要的作用是在代码的基础上开辟一个分叉口，使得代码可以在主干线和分支线同时进行开发，且相互间不会影响。

  5. 删除不需要的分支

与远程版本库协作

git可以与在github的版本库进行协作来达到团队合作开发的目的。

  1. 将Github上的版本库代码下载到本地，假如地址是 https://github.com/example/test.git

  2. 将本地修改的内容同步到远程版本库上，使用push命令，origin 部分指定的是远程版本库的Git地址，master部分指定的是同步到哪一个分支上。

  3. 将远程版本库的修改同步到本地，Git提供了两种命令，fetch和pull
    fetch和push的语法规则类似，执行了这个命令之后会将远程版本库上的代码同步到本地。

  4. 不过同步下来的代码不会合并到任何分支上，会存放到一个origin/master分支上，可以通过diff命令来查看远程版本库上到底修改了哪些东西。

  5. 调用merge命令即可将origin/master的修改合并到主分支上

  6. pull命令则是相当于将fetch和merge命令放在一起执行了，获取最新代码并且合并到本地
