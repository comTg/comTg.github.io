window10和linux双系统开机无法正常引导Linux的解决办法

linux null 2017/08/27

由于bios启动会对windows的efi文件进行保护,所以导致开机后不能正常引导linux的grub,所以只有通过修改windows的efi文件解决了.

  1. 第一步

    用linux系统的grubx64.efi文件去替换Microsoft/Boot/里面的bootmgfw.efi文件,原bootmgfw.efi保留重命名为bootmgfw.efi.old,方便从grub中启动win10,不过如果你不用win10了的话,下面的步骤就不用看了:)

  2. 第二步

    在/etc/grup.d/40_custom文件中手动添加启动文件,也就是之前的bootmgfw.efi.old,格式如下

    hd0值的是sda,也就是第一块硬盘,如果是sdb的话应该改为hd1,sdc也就是hd2,以此类推,gpt2对应sda2或者sdb2,这两个需要你根据自己的硬盘分配来写,用lsdk命令可以查看.

  3. 第三步

    上述两步修改完之后,在终端运行即可.

    sudo update-grub
